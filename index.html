C*Opal: Your AI-Powered CLI Assistant

Interpret natural language commands, execute tasks, and boost your productivity in the terminal with the power of AI.

What is C*Opal?

C*Opal is an AI-powered command-line interface (CLI) assistant designed to streamline your workflow by allowing you to interact with your terminal using natural language. Instead of memorizing complex commands, you can simply tell C*Opal what you want to achieve, and it will generate and execute the appropriate commands for you.

How it Works

C*Opal is a CLI application that uses AI to process natural language commands and execute them in the terminal. It works by taking your natural language input, sending it to a configured Large Language Model (LLM) via your chosen provider. The LLM interprets your request, and if necessary, uses available tools to gather information or perform tasks. Finally, the LLM generates the appropriate terminal command or response, which C*Opal executes and presents the result.

Usage

Non-interactive mode

In non-interactive mode, the AI converts your natural language request into a single terminal command and executes it:

copal "your command in natural language"

Examples:

copal "list all files in the current directory"
copal "delete all files in current folder"

When using a potentially destructive command, the application will ask for confirmation before executing.

Interactive mode

In interactive mode, the AI maintains a continuous conversation, suggesting and executing commands with your permission:

copal --interactive "your task description"

Or use the shorthand:

copal -i "your task description"

The agent will:

  • Analyze your request and suggest appropriate commands
  • Provide reasoning for each suggested command
  • Ask for confirmation before executing potentially dangerous commands
  • Return command output to the AI for further analysis
  • Maintain conversation context for multi-step tasks

To exit agent mode, type exit or quit.

Examples:

copal -i "help me find large files on my system"
copal -i "set up a basic Node.js project"

Advanced Usage: Using with Pipes

The AI CLI can process input from other commands via pipes:

# Process file contents
cat config.json | copal "validate this JSON file"

# Filter command output
ps aux | copal "find all Node.js processes"

# Analyze logs
tail -n 100 server.log | copal "what errors occurred in the last 100 lines"

Thread Management

Threads allow you to maintain conversational context across multiple interactions with the AI. This is especially useful for complex tasks that require multiple steps or when you want to continue a conversation later.

Thread Basics

Each thread maintains its conversation history, allowing the AI to reference previous commands and responses. This provides continuity and context awareness for ongoing tasks.

Managing Threads

Listing Threads:

copal thread list

You can filter threads by name:

copal thread list --filter "project"

After listing threads, you can select one interactively to continue the conversation.

Attaching to a Thread:

copal thread attach <thread-id>

This will resume the conversation where you left off, with all previous context intact.

Using a Thread for a Single Command:

copal --thread <thread-id> "your command"

Renaming Threads:

copal thread rename <thread-id> "React project setup"

Deleting Threads:

copal thread delete <thread-id>

Thread Workflow Example

1. Start a new task in agent mode:

copal -i "set up a React project"

This creates a new thread automatically.

2. Later, list your threads to find the React project:

copal thread list

3. Attach to the thread to continue the setup:

copal thread attach <thread-id>

4. After completing the task, rename the thread for future reference:

copal thread rename <thread-id> "React project setup"

Profile Management

Profiles allow you to manage multiple AI provider configurations, making it easy to switch between different providers or models without reconfiguring your settings each time.

Profile Basics

Each profile contains:

  • A unique name
  • An AI provider (OpenAI, Gemini, VertexAI, or Ollama)
  • An API key (where applicable)
  • A selected model

The active profile determines which AI provider and model is used for processing your commands.

Managing Profiles

Listing Profiles:

copal profile list

This shows all profiles with their provider, model, and pricing information. The active profile is marked.

Setting the Active Profile:

copal profile set <profile-name>

All subsequent commands will use this profile until you change it again.

Deleting a Profile:

copal profile delete <profile-name>

If you delete the active profile, another profile will automatically be set as active.

Viewing Available Models:

copal profile models

You can filter models by provider:

copal profile models --provider openai

Using a Specific Profile for a Command

You can temporarily use a different profile for a single command:

copal --profile <profile-name> "your command"

Or use the shorthand:

copal -p <profile-name> "your command"

Profile Setup Example

1. Run the setup wizard to create your first profile:

copal setup

2. Create additional profiles for different providers:

# The setup command will guide you through creating a new profile
copal setup

3. Switch between profiles as needed:

copal profile set work-openai
copal profile set personal-gemini

4. Use a specific profile without switching:

copal -p work-openai "analyze this log file"

Cost Tracking

The application tracks approximate costs for API usage based on token consumption. This gives you an idea of the cost of your commands across different providers and models.

Note that cost estimates may not be completely accurate but provide a helpful reference for managing your API usage.

Getting Started

Ready to enhance your terminal experience with AI? Follow these simple steps to get C*Opal up and running:

Installation

Install C*Opal globally using npm:

npm install -g copal

Or using yarn:

yarn global add copal

Setup

If you are using Ollama, make sure the server is running and the llama3.2 model is downloaded.

Run the initialization command to configure your AI provider:

copal setup

Follow the prompts to enter your API keys and preferences. The configuration will be stored in ~/.terminal-ai/config.yaml.
